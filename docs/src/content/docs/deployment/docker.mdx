Docker Compose is the recommended way to run Clawbolt. It handles the app and tunnel setup.

## Quick start

```bash
# Clone and configure
git clone https://github.com/mozilla-ai/clawbolt.git
cd clawbolt
cp .env.example .env
# Edit .env with your credentials

# Start everything
docker compose up --build
```

## What Docker Compose starts

| Service | Description |
|---------|-------------|
| **app** | FastAPI server on port 8000, with ffmpeg for audio |
| **tunnel** | Cloudflare Tunnel for HTTPS webhook registration |

On startup, the app:
1. Creates the `data/` directory for file-based storage
2. Starts the FastAPI server
3. Auto-registers the Telegram webhook via the Cloudflare Tunnel URL

## Data persistence

All data is stored as files under the `data/` directory. By default, Docker Compose bind-mounts `./data` from your host into the container at `/app/data`. Data persists across container restarts.

```bash
# View stored data
ls data/users/
```

## Verify it's running

```bash
curl http://localhost:8000/api/health
# {"status":"ok"}
```

## Troubleshooting

### Docker build fails with dependency errors

Some optional dependencies require specific system libraries. The Dockerfile includes `ffmpeg` for audio processing.

```bash
# Rebuild without cache
docker compose build --no-cache
```

### Port 8000 already in use

```bash
# Stop existing containers
docker compose down

# Or use a different port
docker compose up --build -e APP_PORT=8080
```

### Reset all data

```bash
# Remove all user data
rm -rf data/users/
docker compose up --build
```
